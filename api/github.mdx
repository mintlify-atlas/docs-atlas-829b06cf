## OAuth Flow

### Get Authorization URL

**GET** `/api/v1/github/auth-url`

Returns GitHub OAuth authorization URL and signed state token.

<ResponseField name="url" type="string">
  GitHub OAuth URL
</ResponseField>

<ResponseField name="state" type="string">
  Signed state token (expires in 10 minutes)
</ResponseField>

---

### OAuth Callback

**POST** `/api/v1/github/callback`

Handles OAuth callback and creates/updates connection.

<ParamField body="code" type="string" required>
  OAuth authorization code
</ParamField>

<ParamField body="state" type="string" required>
  Signed state token
</ParamField>

<ResponseField name="status" type="string">
  Connection status
</ResponseField>

<ResponseField name="connection_id" type="string">
  GitHub connection ID
</ResponseField>

<ResponseField name="username" type="string">
  GitHub username
</ResponseField>

---

## Connections

### List Connections

**GET** `/api/v1/github/connections`

Returns your active GitHub connections.

<ResponseField name="connections" type="array">
  Array of connection objects
</ResponseField>

---

### Sync Repositories

**POST** `/api/v1/github/sync/{connection_id}`

Synchronizes repositories for a connection.

<ResponseField name="status" type="string">
  Sync status
</ResponseField>

<ResponseField name="synced_count" type="integer">
  Number of repositories synced
</ResponseField>

## Repositories

### List Repositories

**GET** `/api/v1/github/repositories/{connection_id}`

Returns repositories for a connection.

<ResponseField name="repositories" type="array">
  Array of repository objects
</ResponseField>

---

### Analyze Repository

**POST** `/api/v1/github/analyze/{connection_id}/{repo_id}`

Analyzes a repository to detect site type and build configuration.

<ResponseField name="id" type="string">
  Repository ID
</ResponseField>

<ResponseField name="full_name" type="string">
  Repository full name (owner/repo)
</ResponseField>

<ResponseField name="site_type" type="string">
  Detected site type (nextjs, docusaurus, mintlify, etc.)
</ResponseField>

<ResponseField name="build_command" type="string">
  Detected build command
</ResponseField>

<ResponseField name="output_dir" type="string">
  Detected output directory
</ResponseField>

## Pull Requests

</ParamField>

<ParamField body="repo_id" type="string" required>

</ParamField>

<ParamField body="audit_id" type="integer" required>
  Audit ID
</ParamField>

<ParamField body="fixes" type="array" required>
  Array of fix objects
</ParamField>

<ResponseField name="id" type="string">
  Pull request ID
</ResponseField>

<ResponseField name="pr_number" type="integer">
  GitHub PR number
</ResponseField>

<ResponseField name="title" type="string">
  PR title
</ResponseField>

<ResponseField name="html_url" type="string">
  GitHub PR URL

<ResponseField name="status" type="string">
  PR status
</ResponseField>

## Blog Auditing

### Audit Repository Blogs

**POST** `/api/v1/github/audit-blogs/{connection_id}/{repo_id}`

Audits all blog posts in a repository.

<ResponseField name="total_blogs" type="integer">
  Total blogs found
</ResponseField>

<ResponseField name="blogs_audited" type="integer">
  Blogs successfully audited
</ResponseField>

<ResponseField name="results" type="array">
  Audit results for each blog
</ResponseField>

---

### Create Blog Fixes PR

**POST** `/api/v1/github/create-blog-fixes-pr/{connection_id}/{repo_id}`

<ParamField body="blog_paths" type="array" required>
  Array of blog file paths to fix
</ParamField>

<ResponseField name="status" type="string">

</ResponseField>

<ResponseField name="pr" type="object">
  Created PR object
</ResponseField>

<ResponseField name="fixes_applied" type="integer">
  Number of fixes applied
</ResponseField>

<ResponseField name="blogs_fixed" type="integer">
  Number of blogs fixed
</ResponseField>

## Fix Inputs

### Get Fix Inputs

**GET** `/api/v1/github/fix-inputs/{audit_id}`

Returns missing inputs required to safely apply fixes.

<ResponseField name="audit_id" type="integer">
  Audit ID
</ResponseField>

<ResponseField name="missing_inputs" type="array">
  Array of missing input groups
</ResponseField>

<ResponseField name="missing_required" type="integer">
  Count of required missing inputs
</ResponseField>

---

### Submit Fix Inputs

**POST** `/api/v1/github/fix-inputs/{audit_id}`

Submits user-provided fix inputs.

<ParamField body="inputs" type="array" required>
  Array of input answers
</ParamField>

---

### Fix Inputs Chat

**POST** `/api/v1/github/fix-inputs/chat/{audit_id}`

Get LLM-guided suggestions for missing fix inputs.

<ParamField body="issue_code" type="string" required>
  Issue code
</ParamField>

<ParamField body="field_key" type="string" required>
  Field key
</ParamField>

<ParamField body="field_label" type="string">
  Field label
</ParamField>

<ResponseField name="assistant_message" type="string">
  AI assistant message
</ResponseField>

<ResponseField name="suggested_value" type="string">
  Suggested value based on audit evidence
</ResponseField>

<ResponseField name="confidence" type="string">
  Confidence level (evidence, unknown)
</ResponseField>

## Error Codes

<ResponseField name="400" type="error">
  Invalid OAuth callback or missing parameters
</ResponseField>

<ResponseField name="401" type="error">
  Invalid OAuth state or webhook signature
</ResponseField>

<ResponseField name="403" type="error">
  Cross-user access denied
</ResponseField>

<ResponseField name="422" type="error">
  Missing required fix inputs
</ResponseField>
